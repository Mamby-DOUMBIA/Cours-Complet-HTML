Accessibilité Web

Rendez vos pages Web accessibles à tous : personnes malvoyantes, malentendantes, motrices limitées. Maîtrisez les attributs ARIA, les bonnes pratiques WCAG et testez l'accessibilité de vos pages.

11.1 — Pourquoi l'accessibilité est importante

Environ 15% de la population mondiale vit avec un handicap (OMS). L'accessibilité Web (souvent abrégée a11y) consiste à concevoir des pages utilisables par tous, quel que soit leur handicap : visuel, auditif, moteur ou cognitif.

L'accessibilité n'est pas seulement une question éthique : dans de nombreux pays, elle est obligatoire légalement pour les sites des administrations publiques.

11.2 — Les attributs ARIA

ARIA (Accessible Rich Internet Applications) est un ensemble d'attributs HTML qui permettent de donner des informations supplémentaires aux technologies d'assistance (lecteurs d'écran).

Attribut ARIAUsageExemple
aria-labelNom accessible pour un élément sans texte visible<button aria-label="Fermer la modal">✕</button>
aria-labelledbyRéférence à l'id d'un élément qui nomme ce composant<section aria-labelledby="titre-section">
aria-describedbyRéférence à un élément qui décrit ce composant<input aria-describedby="aide-email">
aria-hidden="true"Masque l'élément aux lecteurs d'écranIcônes décoratives
aria-expandedÉtat ouvert/fermé d'un composantaria-expanded="false"
aria-requiredChamp de formulaire obligatoirearia-required="true"
roleRôle sémantique du composantrole="alert", role="dialog"
aria-liveAnnonce les changements de contenuaria-live="polite"
Exemples ARIA pratiquesHTML
<!-- Navigation avec aria-label pour la différencier d'autres nav -->
<nav aria-label="Navigation principale">...</nav>
<nav aria-label="Fil d'Ariane">...</nav>

<!-- Icône décorative masquée + texte lisible -->
<button type="button">
  <span aria-hidden="true">🔍</span>
  <span>Rechercher</span>
</button>

<!-- Champ avec aide contextuelle -->
<label for="mdp">Mot de passe</label>
<input type="password" id="mdp" aria-describedby="aide-mdp">
<p id="aide-mdp">8 caractères min, 1 majuscule, 1 chiffre</p>

<!-- Skip link : lien pour sauter au contenu (crucial pour clavier) -->
<a href="#contenu-principal" class="skip-link">
  Aller au contenu principal
</a>

11.3 — Checklist d'accessibilité

✅ Checklist accessibilité HTML
  • Tout <img> a un attribut alt (même vide pour les images décoratives)
  • Tous les champs de formulaire ont un <label> associé via for/id
  • L'ordre des titres est logique (h1 → h2 → h3, sans sauter)
  • Les liens ont un texte descriptif (pas "cliquez ici")
  • La page est navigable au clavier (Tab pour avancer, Shift+Tab pour reculer)
  • Les éléments focusables ont un style de focus visible (pas de outline:none sans remplacement)
  • Le contraste texte/fond respecte WCAG 2.1 AA (ratio 4.5:1 minimum)
  • La langue est déclarée sur <html lang="fr">
  • Un "skip link" permet de sauter la navigation
  • Les vidéos ont des sous-titres
✏️ Exercice 11.1 — Audit d'accessibilité

Prenez une de vos pages HTML précédentes et auditez-la : (1) Installez l'extension axe DevTools dans Chrome ou Firefox, (2) Ouvrez la page, lancez l'audit, (3) Corrigez les 5 premiers problèmes signalés, (4) Testez la navigation au clavier (Tab uniquement, sans souris).
