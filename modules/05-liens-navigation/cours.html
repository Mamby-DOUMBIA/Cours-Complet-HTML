Liens et Navigation

Créez des liens hypertextes internes, externes, des ancres et des liens spéciaux (email, téléphone). Comprenez la sécurité des liens et construisez une navigation professionnelle.

5.1 — La balise <a> : créer des liens

La balise <a> (anchor = ancre) crée des liens hypertextes. C'est l'élément fondamental du Web : c'est grâce aux liens que les pages sont connectées entre elles.

Anatomie complète d'un lienHTML
<!-- Lien externe simple -->
<a href="https://developer.mozilla.org/fr/">Visiter MDN</a>

<!-- Lien avec tous les attributs importants -->
<a
  href="https://developer.mozilla.org/fr/"
  target="_blank"
  rel="noopener noreferrer"
  title="Documentation officielle HTML - MDN"
>
  Documentation MDN (s'ouvre dans un nouvel onglet)
</a>
AttributValeurs possiblesDescription
hrefURL, chemin, #id, mailto:, tel:Destination du lien. Obligatoire pour qu'un lien fonctionne.
target_blank, _self, _parent, _topOù ouvrir le lien. _blank = nouvel onglet, _self = même onglet (défaut).
relnoopener noreferrer, nofollowRelation avec la destination. Sécurité et SEO.
titleTexteInfo-bulle au survol. Aide à l'accessibilité.
downloadNom de fichier ou videForce le téléchargement du fichier lié.
hreflangCode langue (fr, en…)Langue de la page de destination.

5.2 — Liens externes : sécurité avec noopener noreferrer

Quand vous ouvrez un lien dans un nouvel onglet avec target="_blank", la page de destination peut accéder à votre page via window.opener. C'est une faille de sécurité appelée reverse tabnapping.

La solution : toujours ajouter rel="noopener noreferrer" avec target="_blank" :

Lien externe sécuriséHTML
<!-- ✅ Lien externe sécurisé -->
<a href="https://exemple.com" target="_blank" rel="noopener noreferrer">Lien sécurisé</a>

<!-- ❌ Lien dangereux (pas de rel) -->
<a href="https://exemple.com" target="_blank">Lien non sécurisé</a>

<!-- noopener : empêche l'accès à window.opener
     noreferrer : n'envoie pas la page de provenance (Referer) -->

5.3 — Liens internes : chemins relatifs

Un lien interne pointe vers une autre page de votre propre site. On utilise des chemins relatifs (relatifs à la position du fichier actuel).

Chemins relatifs selon la structure du projetHTML
<!-- Structure du projet :
mon-site/
├── index.html
├── contact.html
├── blog/
│   ├── article-1.html
│   └── article-2.html
└── images/
    └── photo.jpg
-->

<!-- Depuis index.html -->
<a href="contact.html">Contact</a>           <!-- même dossier -->
<a href="blog/article-1.html">Article</a>   <!-- sous-dossier -->

<!-- Depuis blog/article-1.html -->
<a href="../index.html">Accueil</a>        <!-- dossier parent -->
<a href="../contact.html">Contact</a>     <!-- dossier parent -->
<a href="article-2.html">Article 2</a>  <!-- même dossier -->

5.4 — Les ancres : liens vers une section de page

Une ancre permet de créer un lien vers une section spécifique d'une page (comme la table des matières d'un long article). Cela fonctionne en deux étapes :

  1. Ajouter un id à l'élément de destination
  2. Créer un lien avec href="#id"
Liens avec ancresHTML
<!-- Table des matières (en haut de page) -->
<nav>
  <a href="#introduction">Introduction</a>
  <a href="#installation">Installation</a>
  <a href="#conclusion">Conclusion</a>
</nav>

<!-- Sections cibles (avec id correspondant) -->
<h2 id="introduction">Introduction</h2>
<p>...</p>

<h2 id="installation">Installation</h2>
<p>...</p>

<!-- Lien ancre vers une autre page -->
<a href="guide.html#installation">Voir l'installation</a>

5.5 — Liens spéciaux : email, téléphone, téléchargement

Types de liens spéciauxHTML
<!-- Lien email : ouvre le client mail avec destinataire pré-rempli -->
<a href="mailto:contact@exemple.com">Envoyer un email</a>

<!-- Avec sujet et corps pré-remplis -->
<a href="mailto:contact@exemple.com?subject=Demande&body=Bonjour">
  Email avec sujet
</a>

<!-- Lien téléphone : sur mobile, lance l'appel directement -->
<a href="tel:+33123456789">+33 1 23 45 67 89</a>

<!-- Lien de téléchargement (download) -->
<a href="fichiers/cv.pdf" download="CV-Jean-Dupont.pdf">
  Télécharger mon CV (PDF)
</a>

<!-- Lien placeholder (ne va nulle part) -->
<a href="#">Lien non encore actif</a>

5.6 — La balise <nav> et les bonnes pratiques

Navigation professionnelleHTML
<!-- Navigation principale du site -->
<nav aria-label="Navigation principale">
  <ul>
    <li><a href="index.html">Accueil</a></li>
    <li><a href="services.html">Services</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>
✅ Bonnes pratiques pour les liens
  • Le texte d'un lien doit être descriptif : évitez "Cliquez ici" ou "En savoir plus". Préférez "Télécharger le rapport annuel 2024".
  • Les liens externes ouverts dans un nouvel onglet doivent toujours avoir rel="noopener noreferrer".
  • Un lien doit rester reconnaissable : il doit être visuellement distinct (souligné, couleur différente).

5.7 — Exercices pratiques

✏️ Exercice 5.1 — Navigation multi-pages

Créez 3 fichiers HTML : index.html, a-propos.html et contact.html. Chaque page doit avoir une barre de navigation avec des liens vers les 2 autres pages. Testez que tous les liens fonctionnent dans les deux sens.

✏️ Exercice 5.2 — Table des matières avec ancres

Créez un long article avec 4 sections (Introduction, Développement, Exemples, Conclusion). En haut de la page, créez une table des matières avec des liens ancres vers chaque section. Ajoutez aussi un lien "Retour en haut" à la fin de chaque section.
