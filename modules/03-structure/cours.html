Structure d'un document HTML

Découvrez l'anatomie complète d'une page HTML valide : DOCTYPE, html, head, body. La base que tout développeur doit maîtriser parfaitement.

3.1 — La structure minimale obligatoire

Tout fichier HTML valide doit respecter une structure précise. Voici la structure complète commentée :

Structure HTML5 complèteHTML
<!DOCTYPE html>                    <!-- Déclare HTML5 -->
<html lang="fr">                    <!-- Balise racine, langue française -->
<head>
  <meta charset="UTF-8">            <!-- Encodage — TOUJOURS en premier -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Description de la page (155 car. max.)">
  <title>Titre de la page — Nom du site</title>
  <link rel="stylesheet" href="css/style.css">
</head>
<body>
  <h1>Titre principal</h1>
  <p>Premier paragraphe.</p>
  <script src="js/script.js"></script>  <!-- JS en bas du body -->
</body>
</html>

3.2 — La déclaration DOCTYPE

<!DOCTYPE html> doit être la toute première ligne du fichier. Elle indique au navigateur d'utiliser le standards mode (HTML5 moderne) et non le "quirks mode" des vieux navigateurs. Sans DOCTYPE, votre page peut s'afficher de manière incorrecte.

📜 Évolution du DOCTYPE

En HTML4, le DOCTYPE était interminable : <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">. HTML5 l'a simplifié à son minimum absolu : <!DOCTYPE html>.

3.3 — L'élément <html lang>

La balise racine <html> englobe tout le document. L'attribut lang déclare la langue du contenu. Il est essentiel pour :

  • Les lecteurs d'écran : ils prononcent le texte avec la bonne prononciation.
  • Les moteurs de recherche : ils indexent votre page dans la bonne région linguistique.
  • La traduction automatique : les navigateurs détectent automatiquement la langue.
Codes de langue courantsHTML
<html lang="fr">      <!-- Français -->
<html lang="fr-BE">   <!-- Français de Belgique -->
<html lang="en">      <!-- Anglais -->
<html lang="ar" dir="rtl">  <!-- Arabe, lecture droite→gauche -->

3.4 — Le <head> en détail

Le <head> est invisible à l'écran, mais lu par les navigateurs, les robots d'indexation et les réseaux sociaux. Voici les éléments indispensables :

ÉlémentObligatoire ?Rôle
<meta charset="UTF-8">✅ OuiEncodage des caractères. Supporte accents, emojis, etc.
<meta name="viewport">✅ OuiAffichage mobile correct. Sans ça, les mobiles zooment à 980px.
<title>✅ OuiTitre dans l'onglet et Google. 50–60 caractères idéalement.
<meta name="description">RecommandéRésumé pour Google. Affiché sous le titre dans les résultats. 155 car.
<link rel="stylesheet">Si CSSLien vers la feuille de style CSS externe.
<meta name="author">NonNom de l'auteur de la page.
<link rel="icon">NonFavicon (petite icône dans l'onglet).

3.5 — Le <body> : tout ce qui est visible

Le <body> contient 100% du contenu visible : textes, images, liens, formulaires, vidéos… Il est unique dans chaque page HTML.

💡 Bonne pratique : scripts en bas du body

Placez toujours vos balises <script> juste avant </body>. Ainsi, le HTML s'affiche d'abord, et le JavaScript se charge ensuite, ce qui améliore la vitesse de chargement perçue.

3.6 — L'indentation : lisibilité du code

❌ Sans indentation
<html><head><title>Site</title></head><body><h1>Titre</h1><p>Texte</p></body></html>
✅ Avec indentation (2 espaces)
<html>
  <head>
    <title>Site</title>
  </head>
  <body>
    <h1>Titre</h1>
    <p>Texte</p>
  </body>
</html>

3.7 — Exercices pratiques

✏️ Exercice 3.1 — Structure de base

Créez un fichier ma-page.html avec une structure HTML5 complète et valide. Elle doit contenir : le DOCTYPE, la balise html avec lang="fr", un head avec charset, viewport, title et description, et un body avec un h1 et un paragraphe. Validez votre page sur validator.w3.org.

✏️ Exercice 3.2 — Identifier les erreurs

Le code ci-dessous contient 4 erreurs. Trouvez-les et corrigez-les :

Code à corrigerHTML
<html>
<head>
  <title>Ma page</title>
  <meta charset="UTF-8">
</head>
<body>
  <h1>Bonjour
  <p>Bienvenue sur mon site.
<body>
✅ Corrigé — Exercice 3.2

Les 4 erreurs : (1) DOCTYPE manquant, (2) lang manquant sur <html>, (3) <h1> non fermé, (4) </body> écrit <body> (manque le slash) et </html> absent.

Code corrigéHTML
<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <title>Ma page</title>
</head>
<body>
  <h1>Bonjour</h1>
  <p>Bienvenue sur mon site.</p>
</body>
</html>
