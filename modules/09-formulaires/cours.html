Formulaires HTML

Les formulaires HTML permettent aux utilisateurs de saisir et envoyer des données. Maîtrisez la balise form, tous les types d'input, la validation native HTML5, et les bonnes pratiques d'accessibilité.

9.1 — La balise <form>

La balise <form> est le conteneur de tout formulaire. Ses attributs définissent comment les données seront envoyées au serveur.

Structure d'un formulaireHTML
<form
  action="traitement.php"
  method="post"
  enctype="multipart/form-data"
  novalidate
>
  <!-- Les champs du formulaire vont ici -->
</form>

<!-- action : URL du serveur qui traite le formulaire
     method : "get" (données dans l'URL) ou "post" (données cachées — préféré)
     enctype : nécessaire pour les uploads de fichiers
     novalidate : désactive la validation HTML5 native (utile pour tester) -->

9.2 — Les champs <input>

La balise <input> est la plus polyvalente des balises de formulaire. Son comportement change complètement selon son attribut type.

typeRenduUsage
textChamp texte une ligneNom, prénom, ville
emailChamp texte + validation emailAdresse email
passwordChamp texte masqué (●●●●)Mot de passe
numberChamp numérique avec flèchesÂge, quantité
telChamp texte + clavier numérique mobileTéléphone
urlChamp texte + validation URLSite Web
dateSélecteur de dateDate de naissance
timeSélecteur d'heureHeure de rendez-vous
checkboxCase à cocherOptions multiples
radioBouton radioChoix unique parmi plusieurs
rangeCurseur glissantVolume, intensité
fileSélecteur de fichierUpload d'image, de document
colorSélecteur de couleurPersonnalisation de couleur
searchChamp de rechercheBarre de recherche
hiddenInvisible (valeur cachée)Données techniques non saisies
submitBouton "Envoyer"Soumettre le formulaire
resetBouton "Réinitialiser"Vider le formulaire
Formulaire complet avec labels et validationHTML
<form action="/inscription" method="post">

  <!-- Champ texte avec label associé -->
  <label for="prenom">Prénom *</label>
  <input
    type="text"
    id="prenom"
    name="prenom"
    required
    placeholder="Entrez votre prénom"
    minlength="2"
    maxlength="50"
    autocomplete="given-name"
  >

  <!-- Email avec validation automatique -->
  <label for="email">Email *</label>
  <input
    type="email"
    id="email"
    name="email"
    required
    placeholder="vous@exemple.com"
    autocomplete="email"
  >

  <!-- Boutons radio (choix unique) -->
  <fieldset>
    <legend>Niveau d'expérience *</legend>
    <label><input type="radio" name="niveau" value="debutant" required> Débutant</label>
    <label><input type="radio" name="niveau" value="intermediaire"> Intermédiaire</label>
    <label><input type="radio" name="niveau" value="expert"> Expert</label>
  </fieldset>

  <!-- Textarea pour texte long -->
  <label for="message">Message</label>
  <textarea
    id="message"
    name="message"
    rows="5"
    placeholder="Votre message..."
    maxlength="1000"
  ></textarea>

  <!-- Liste déroulante -->
  <label for="pays">Pays</label>
  <select id="pays" name="pays">
    <option value="">Sélectionnez votre pays</option>
    <optgroup label="Europe">
      <option value="fr">France</option>
      <option value="be">Belgique</option>
      <option value="ch">Suisse</option>
    </optgroup>
  </select>

  <!-- Case à cocher (CGU) -->
  <label>
    <input type="checkbox" name="cgu" required>
    J'accepte les <a href="cgu.html">conditions générales</a> *
  </label>

  <!-- Bouton de soumission -->
  <button type="submit">S'inscrire</button>

</form>

9.3 — Validation HTML5 native

HTML5 intègre des mécanismes de validation des formulaires sans JavaScript. Ces attributs permettent de vérifier les données côté client avant l'envoi au serveur.

AttributDescriptionExemple
requiredChamp obligatoire<input required>
minlengthLongueur minimale en caractèresminlength="8"
maxlengthLongueur maximale en caractèresmaxlength="100"
minValeur minimale (number, date)min="0"
maxValeur maximale (number, date)max="120"
patternExpression régulière de validationpattern="[A-Z]{3}"
type="email"Valide automatiquement le format emailtype="email"
type="url"Valide automatiquement le format URLtype="url"
⚠️ Validation côté serveur indispensable

La validation HTML5 améliore l'expérience utilisateur, mais elle peut être contournée. Toujours valider les données côté serveur en plus de la validation HTML.

✏️ Exercice 9.1 — Formulaire de contact complet

Créez un formulaire de contact avec : Nom (required, min 2 car.), Prénom, Email (required, type email), Téléphone (type tel), Sujet (select avec 5 options), Message (textarea, required, min 20 car.), et un bouton d'envoi. Ajoutez des labels pour chaque champ. Testez la validation en essayant de soumettre le formulaire vide.
