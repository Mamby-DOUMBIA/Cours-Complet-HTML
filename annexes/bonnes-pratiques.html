Bonnes pratiques HTML

✅ Checklist complète — à vérifier avant chaque mise en ligne
  • Structure : DOCTYPE présent, lang sur html, charset UTF-8 en premier dans head
  • Title : unique, 50-60 caractères, mot-clé principal en tête
  • Meta description : unique, 150-160 caractères, incitative
  • Un seul h1 par page, hiérarchie des titres logique
  • Alt sur toutes les images (descriptif pour informatives, vide pour décoratives)
  • Labels associés à tous les champs de formulaire
  • Liens avec texte descriptif, liens externes avec noopener noreferrer
  • width et height sur toutes les images (évite le CLS)
  • loading="lazy" sur les images hors viewport initial
  • Sémantique : header, nav, main, article, section, aside, footer utilisés correctement
  • Scripts en defer ou en bas du body
  • Validation : zéro erreur sur validator.w3.org
  • Responsive : viewport meta, testé sur mobile
  • Open Graph : og:title, og:description, og:image, og:url
  • Favicon : svg + png fallback
  • Accessibilité : navigation au clavier, aria-labels, contraste suffisant

📁 Organisation des fichiers

À faireÀ éviter
Noms de fichiers en minuscules : a-propos.htmlMajuscules : APropos.html
Tirets comme séparateurs : mes-photos.htmlEspaces : mes photos.html
Extension .html : index.htmlExtension .htm (obsolète)
Images dans assets/images/Tout à la racine du projet
CSS dans assets/css/Style dans le HTML (sauf exception)
JS dans assets/js/Script inline sur chaque page
